## Mentions légales / CGU / Confidentialité (#1)

1. Sans être connecté, naviguer vers `/mentions-legales` → contenu markdown rendu en HTML (## titres, listes, gras), date de mise à jour + version visibles

4. Footer : les 3 liens "Politique de confidentialité", "Conditions d'utilisation", "Mentions légales" pointent bien vers les bonnes pages (et non `#`)

### Test L2 — Bandeau cookies
1. Ouvrir le site en navigation privée → bandeau en bas à droite *"Cookies & vie privée"*
2. Cliquer **"J'ai compris"** → bandeau disparaît
3. Recharger la page → bandeau reste fermé (localStorage `immodz-cookies-ack`)
4. Vider le localStorage → bandeau revient au reload

### Test L3 — Checkbox CGU à l'inscription
1. Aller sur `/register`, remplir tous les champs sauf la case CGU
2. **Attendu** : bouton "S'inscrire" désactivé (grisé), submit refusé avec message *"Vous devez accepter les CGU…"*
3. Cocher la case → bouton actif → inscription OK

### Test L4 — Édition admin (la feature clé demandée)
1. Login `admin@immodz.local` → `/admin` → **"Pages légales"** (tuile)
2. Sur `/admin/legal` : 3 lignes (Mentions, CGU, Confidentialité) avec versions et dates
3. Cliquer **"Éditer"** sur CGU
4. **Attendu** : page `/admin/legal/cgu` avec champ titre + textarea markdown + onglet "Aperçu"
5. Modifier le titre + ajouter un paragraphe au markdown
6. Cliquer onglet **"Aperçu"** → rendu visuel mis à jour live
7. Cliquer **"Publier la nouvelle version"** → bandeau vert "Page publiée"
8. Recharger la page admin → version incrémentée (V1 → V2), date mise à jour
9. Aller sur `/cgu` (public, autre onglet) → nouveau contenu visible

### Test L5 — Sécurité admin

2. Login non-admin → tenter `POST /api/admin/legal` (curl) → 403

### Test L6 — Validation

## Signalement d'annonce (#2)

### Test R1 — Bouton sur la fiche annonce
1. Sur n'importe quelle annonce active, en bas du formulaire de contact, **bouton "Signaler"** (gris, icône drapeau)
2. Cliquer → modal s'ouvre

### Test R2 — Signalement non connecté
1. Déconnecté → cliquer "Signaler" sur une annonce
2. **Attendu** : modal affiche *"Connectez-vous pour signaler une annonce"* + bouton "Se connecter" (avec callback URL)

### Test R3 — Signalement nominal
1. Login `user1@immodz.local`, ouvrir une annonce **qui n'est pas la sienne**
2. "Signaler" → modal → choisir motif "Annonce frauduleuse / faux bien"
3. Ajouter un commentaire *"La photo est volée d'un autre site"*
4. **"Signaler"** → bandeau vert *"Merci, votre signalement a été transmis"*
5. **Vérifier en BDD** : 1 row `listing_reports` avec status `PENDING`

### Test R4 — Auto-signalement bloqué
1. Login `director@immodz.local`, ouvrir l'une de **ses propres** annonces
2. Tenter "Signaler" → API renvoie `400 Vous ne pouvez pas signaler votre propre annonce`

### Test R5 — Anti-doublon
1. Refaire R3 avec le même user sur la même annonce
2. **Attendu** : `409 Vous avez déjà signalé cette annonce. Un admin la traite.`

### Test R6 — Rate limit
1. Avec un même user, signaler 5 annonces différentes en moins d'1h → OK
2. 6e tentative → `429 Trop de signalements. Réessayez plus tard.`

### Test R7 — File admin
1. Login `admin@immodz.local` → `/admin` → tuile **"Signalements"** (rouge si > 0)
2. Cliquer → `/admin/reports` : groupé par annonce, chaque signalement liste motif + commentaire + reporter + date
3. Bouton **"Marquer traité"** sur un signalement → disparaît de la liste, status passe à `REVIEWED`
4. Bouton **"Rejeter"** (le signalement, pas l'annonce) → passe à `DISMISSED`
5. Bouton **"Rejeter l'annonce"** → champ motif (min 3 chars) → confirmer
6. **Attendu** :
   - L'annonce passe à `REJECTED` avec le motif (visible dans le dashboard de l'agent)
   - **Tous** les signalements PENDING de cette annonce sont auto-clos en `REVIEWED`
   - L'annonce disparaît de `/recherche` et de la carte

### Test R8 — Sécurité admin
1. Login non-admin → tenter `GET /api/admin/reports` → 403
2. Login non-admin → tenter `POST /api/admin/reports` → 403

### Test R9 — Annonce supprimée pendant le signalement
1. Créer un signalement
2. L'auteur supprime son annonce avant traitement
3. **Attendu** : le signalement est aussi supprimé (FK Cascade) → ne casse pas la file admin
