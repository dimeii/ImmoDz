## Mot de passe oublié (#3b)

### Test M1 — Parcours nominal
1. Sur `/login`, cliquer **"Mot de passe oublié ?"** → page `/mot-de-passe-oublie`
2. Saisir `user1@immodz.local` → "Envoyer le lien"
3. **Attendu UI** : message vert *"Si cet email est associé à un compte, vous recevrez un lien…"*
4. **Attendu Resend** : email reçu avec sujet *"Réinitialisation de votre mot de passe ImmoDz"* + bouton vert
5. Cliquer le bouton dans l'email → page `/reinitialisation/[token]`
6. Saisir nouveau mdp `nouveaupwd1` + confirmation → "Modifier"
7. **Attendu** : message vert + redirection auto vers `/login` après 2s
8. Login avec le nouveau mdp → OK
9. Login avec l'ancien `password123` → "Email ou mot de passe incorrect"

### Test M2 — Anti énumération de comptes
1. `/mot-de-passe-oublie` avec un email qui n'existe pas (`personne@nope.local`)
2. **Attendu** : même message vert qu'en M1 (aucune indication "email inconnu"), pas d'email envoyé
3. **Vérifier** dans la console : pas d'erreur exposée

### Test M3 — Token expiré / déjà utilisé
1. Refaire M1 jusqu'à recevoir l'email
2. Attendre **plus de 60 minutes** (ou en BDD : `UPDATE password_reset_tokens SET "expiresAt" = NOW() - INTERVAL '1 minute' WHERE "tokenHash" = ...`)
3. Cliquer le lien → soumettre nouveau mdp
4. **Attendu** : `400 Lien invalide ou expiré.`
5. Réutiliser le **même token** après l'avoir consommé → idem `400`

### Test M4 — Rate limit
1. Demander un reset 3 fois sur la même IP en moins d'1h
2. 4e tentative → `429 Trop de demandes. Réessayez dans une heure.`

### Test M5 — Invalidation des autres tokens
1. Demander un reset → recevoir email A
2. Sans utiliser A, redemander un reset → recevoir email B
3. Utiliser le lien dans **email B** → reset OK
4. Tenter d'utiliser le lien dans **email A** → `400 Lien invalide ou expiré.`
